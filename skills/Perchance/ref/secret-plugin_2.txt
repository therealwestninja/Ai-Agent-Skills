<h1><span style="display:inline-block; position:relative;">🔑<span style="display:inline-block; position:absolute; right:1rem;">🗝️</span></span> Secret Plugin</h1>
<p style="font-size:80%; opacity:0.7; margin:0 auto; max-width:100%; width:480px;">You can play around with this plugin using <a href="https://perchance.org/public-key-encryption-tool" target="_blank">this interface</a>, and you can make a copy of <a href="https://perchance.org/send-me-a-secret-message" target="_blank">this generator</a> to easily allow anyone to send you a secret message.</p>
<div style="width:800px; max-width:95%; margin:0 auto; text-align:left;">
  
  <p>Imagine that you just befriended someone online in a public chatroom (e.g. a public <a href="/comments-plugin" target="_blank">comments-plugin</a> channel), and they wanted to send you a private/secret message. For example, to share their Steam username so you can play a game together. But the problem is that the chatroom is completely public, and they don't want to tell <i>everyone</i> their Steam username.</p>
  
  <p>It doesn't seem possible for them to send you a secret message, right? Anything recieved by <u>you</u> is recieved by <u>everyone</u>, so how could they possibly send a secret to you? It's surely impossible!?</p>
  
  <p>Hmm, well, what if they could somehow 'encrypt' a message before sending it on the public chat? I.e. turn it into jumbled/encrypted text via a special program/app so it's only reversible if you know the password. And then they could send that encrypted text to you, and then you could use the same program/app to decrypt it back to the real message via the password? Wait, no, that wouldn't work! Because how would they secretly tell you the password? You can only communicate over a completely public chat. <i style="opacity:0.6;">You could of course try to speak in 'code', using in-jokes, or whatever, but that's not full-proof, since someone could scroll up in the chat logs to work out all the subtle hints you're giving. We want a <u>completely</u> full-proof solution.</i></p>
  
  <p>Okay, let's try a new thought experiment: What if there were a special form of data encryption, where you needed to use <b style="color:#4343ff;">one password for encryption</b>, and a <b style="color:#f36500;"><u>separate</u> password for decryption</b>? So you have two different passwords <i style="opacity:0.6;">(which is different to 'normal' encryption where you just use a single password for both)</i>. The two passwords are created via a special process that causes them to be "<a href="https://en.wiktionary.org/wiki/mathemagical" target="_blank">mathemagically</a>" linked - so if some text was encrypted with the encryption password, then <u>only</u> the 'linked' decryption password can be used to unlock it.</p>
  
  <p>Let's think of these two passwords as a pair of magical keys ✨🗝️🔑✨. These magical keys were created via an arcane process that makes them inherently linked to one another. So if you "tap" a message with the "encryption" key, then the message gets jumbled into nonsense-looking random letters and numbers. And the only thing in the known universe that can unjumble it back to the original message is a "tap" from the matching/linked <u>de</u>cryption key.</p>

Answer: You use a 'program' like the one contained in this plugin. You put the key and the text as input, and it'll give you the encrypted/decrypted text as output. You can also use a trusted interface made using this plugin, like <a href="https://perchance.org/public-key-encryption-tool" target="_blank">this one</a>.)</i> -->
  
  <p style="opacity:0.6; font-style:italic;">(Remember, each 'key' is actually just a password - it's just some letters and numbers. But note that you can't decide on your own password text like "cat123" because the process of creating a pair of these 'magically linked' keys requires that each key contains special information. So the keys are very long and random-looking compared to an ordinary password.)</p>
  
  <p>So, let's say you <i>somehow</i> have a pair of these magically linked passwords/keys <i style="opacity:0.6;">(Spoiler: You can make pairs of these keys using this plugin)</i>. Now that we have these two keys, let's come back to our problem: Your new friend wants to send you a secret message, but all your communication with them is completely public.</p>
  
  <p>How could you use these keys to allow your friend to send you a secret message? You can pause here and have a think about it if you want. Even if you don't get the answer, it's helpful to ponder a bit, since our brains learn via 'surprise', and you'll be more surprised by the answer if you've thought for a while and can't see any solution. <i style="opacity:0.6;">(In other words, learning is mostly about <u>effort</u>, and definitely not about "getting questions correct", so don't worry if you can't get it!)</i></p>
  
  <p>...</p>
  
  <p>...</p>
  
  <p>...</p>
  
  <p>Okay, here's the answer: You send the encryption key in the public chat. Everyone can see it, but that's fine. All they can do is <i>encrypt</i> stuff with it. Now your friend has the encryption key, so they can encrypt their secret message, and then send that jumbled/encrypted text in the public chat. It's fine for everyone to see that encrypted text, because they can't decrypt it. You are the only person who has the decryption key, so you're the only one who can "undo" the encryption to read your friend's original message.</p>
  
  <p>Not <i>too</i> complicated, right? As long as you <u>always keep the <b>decryption</b> key private</u>, then everything is fine, and anyone can send you secret messages whenever they want, via any method/channel/etc. - it doesn't matter if they're sending it to you via a public chat, social media post, a shared email account, their nosey cousin's carrier pigeon. As long as they "tapped" their message with your encryption key, then no one else will be able to actually see what it says.</p>
  
  <p>It's really important to keep your <span style="color:#f36500;">decryption key</span> private, so we're actually going to call it the <b style="color:#f36500;"><u>private</u> key</b> from now on - this is the more common term for it in the field of cryptography. And the <span style="color:#4343ff;">encryption key</span> can be made public to everyone, so it's commonly referred to as the <b style="color:#4343ff;"><u>public</u> key</b>. You can learn more about this whole topic on the <a href="https://en.wikipedia.org/wiki/Public-key_cryptography" target="_blank">public-key cryptography</a> Wikipedia page.</p>
  
  <p style="background:#006e8c; color:white; padding:0.5rem; border-radius:3px;">This plugin allows you to generate pairs of these 'magical' keys, and use them to encrypt and decrypt text.</p>
  
  <p>To use this plugin, first import it by putting this line in your Perchance lists code editor:</p>
<pre>
secret = \{import:secret-plugin\}
</pre>
  <p>Here's a minimal demonstration of this plugin, which just (pointlessly) encrypts some text and then decrypts it:</p>
<pre>
let keys = secret.generateKeyPair(); <span style="opacity:0.5;">// generate pair of keys \{public, private\}</span>
<span style="opacity:0.5;">// now we can write `keys.public` to access the public key/password, and `keys.private` to access the private key/password</span>
let encrypted = secret.encrypt("Hello!", keys.public); <span style="opacity:0.5;">// encrypt some text with public key</span>
<span style="opacity:0.5;">// the `encrypted` variable now contains some text like "ENCRYPTED_1_kLSjgGhsEb064jv......XVtIF0ZSacGYdW_ENCRYPTED_END"</span>
let decrypted = secret.decrypt(encrypted, keys.private); <span style="opacity:0.5;">// decrypt the text with private key</span>
<span style="opacity:0.5;">// the `decrypted` variable now contains the text "Hello!"</span>
</pre>
  <p>And now a slightly larger example with more explanation:</p>
<pre>
&lt;pre id="outputEl" style="text-align:left;">&lt;/pre>
&lt;script>
  <span style="opacity:0.5;">// first we generate the two 'magical' keys</span>
  <span style="opacity:0.5;">// the 'public' key is the one that encrypts things, and the 'private' key is the one that decrypts them</span>
  <span style="opacity:0.5;">// these are both just text. they're like passwords, except we have separate passwords for encryption and decryption</span>
  let keys = secret.generateKeyPair();
  <span style="opacity:0.5;">// now we create the message that we want to encrypt:</span>
  let originalText = "The text to be encrypted.";
  <span style="opacity:0.5;">// now we encrypt it using the public key:</span>
  let encryptedText = secret.encrypt(originalText, keys.public);
  <span style="opacity:0.5;">// and now let's check that if we decrypt it (with the private key), we get the original message:</span>
  let decryptedText = secret.decrypt(encryptedText, keys.private);
  <span style="opacity:0.5;">// output all the variables so we can check the result:</span>
  outputEl.innerHTML = JSON.stringify(\{publicKey:keys.public, privateKey:keys.private, originalText, encryptedText, encryptedTextLength:encryptedText.length, decryptedText, success: originalText === decryptedText\}, null, 2);
&lt;/script>
</pre>
  
  <p><a href="https://perchance.org/secret-plugin-example#edit">Here's an example generator</a> which uses code similar to above.</p>
  
  <p>This plugin isn't inherently linked to Perchance in any fundamental sense. If, for example, you want to send an email to your friend, but you know that your email provider may snoop on your emails, then you can ask your friend to generate a key pair using this plugin and get them to send you their <u>public</u> key (via email - since it's completely fine if anyone sees anyone's public key - even the snoopers). Then you can use this plugin (e.g. via <a href="https://perchance.org/public-key-encryption-tool" target="_blank">this simple interface I made</a>) to encrypt the message with their public key, and send them the resulting garbled text. Then when they recieve it, they'd use that same interface to decrypt it using their <u>private</u> key.</p>
  
  <p>If you want to use this plugin to encrypt/decrypt an image, or any other type of file, then you'd first convert the image into a <a href="https://developer.mozilla.org/en-US/docs/Web/URI/Schemes/data" target="_blank">data URL</a> (i.e. a text representation of the image/file).</p>
  
  <p><a href="https://perchance.org/send-me-a-secret-message" target="_blank">Here's an example generator</a> which allows people to send you private messages using the comments plugin. It uses the <code>beforeSubmit</code> option of the comments plugin to encrypt their message with your public key just before it is submitted.</p>
  
  <p style="font-weight:bold;">Notes:</p> 
  <ul>
    <li style="margin-top:0.5rem;">You can also use this plugin to encrypt your own personal data that you don't intend to send to anyone. I.e. this plugin can be used for more than just for private communication with others. If you just want to use it for encryption of your private files/info, then just generate a key pair, and don't share the public key with anyone. You can just encrypt stuff with your 'public' key and decrypt it with your private key. It's like normal encryption with a password, except you have a separate password for encryption and decryption, instead of just one password for both. Even in this case, it's still fine if others see your public key, since all they can do is <u>en</u>crypt stuff with it.</li>
    <li style="margin-top:0.5rem;">Why did I build this? Well, I often get feedback on my generators that I can't reproduce - i.e. the error that they're talking about doesn't happen for me. Almost always it has to do with some weird Safari bug that only occurs in specific browser versions, or something like that. So I wanted to make it so when the user submits feedback, it appends their browser version to their feedback message, using the <a href="/comments-plugin" target="_blank">comments-plugin</a>'s <code>beforeSubmit</code> option. But that's a bit of a privacy issue, since comments plugin messages are public, and people might not want <i>everyone</i> to know which browser they're using. So I needed something to encrypt it, in such a way that a "secret" password was only needed for <i>decryption</i>, and not encryption. That eventually led to the creation of this plugin. <a href="https://perchance.org/bug-report-plugin-encryption-example#edit" target="_blank">Here's a demo</a> which uses this plugin for the above-mentioned browser debug info collection. And <a href="https://perchance.org/bug-report-plugin-encryption-simpler-example#edit" target="_blank">here's a simpler version</a>.</li>
    <li style="margin-top:0.5rem;">You'll notice that encrypting a small amount of text produces a long string. That's because there's basically a minimum size for the encryption. You can then try a string that's a bit longer, and you'll notice that the garbled text is basically the same length. Once you put a really long string, you'll start to see the encrypted text size start to increase.</li>
    <li style="margin-top:0.5rem;">Many encryption methods are susceptible to being cracked by hypothetical quantum computers that may be built in the next decade or two. The encryption scheme used by this plugin (<a href="https://github.com/dajiaji/crystals-kyber-js/tree/2.3.0" target="_blank">ML-KEM FIPS 203</a>) is a modern one that cannot (according to current theoretical knowledge) be broken by quantum computers.</li>
    <li style="margin-top:0.5rem;">This plugin automatically compresses the text you input before it's encrypted, and decompresses during decryption, so if you're e.g. encrypting repetitive text, there's no need to do the compression yourself - it's already done for you.</li>
    <li style="margin-top:0.5rem;">When copying around public/private keys, you can double-click the key (or long-press it on mobile) as a fast way to select the whole thing. This works because keys never contain spaces or special characters, so the "double-click or long-press to select the whole word" shortcut that is common across operating systems works to select the whole key (i.e. the key is effectively treated as a single word). It's easier and less error-prone than clicking and dragging to highlight it. The same applies to encrypted messages, by the way - they don't have spaces or special characters.</li>
    <li style="margin-top:0.5rem;">Public keys always start with <span style="opacity:0.6;">PUBLIC_</span> and end with <span style="opacity:0.6;">_PUBLIC_END</span>. And private keys always start with <span style="opacity:0.6;">PRIVATE_</span> and end with <span style="opacity:0.6;">_PRIVATE_END</span>. So you can easily double-check that you got the whole thing after copy-pasting a key somewhere. Also, encrypted messages start with <span style="opacity:0.6;">ENCRYPTED_</span> and end with <span style="opacity:0.6;">_ENCRYPTED_END</span>.</li>
    <li style="margin-top:0.5rem;">You may have noticed there's a number after <span style="opacity:0.6;">PUBLIC_</span> and <span style="opacity:0.6;">PRIVATE_</span> and <span style="opacity:0.6;">ENCRYPTED_</span>. That's a version number. It allows the underlying encryption scheme to get updated, while ensuring that this plugin continues to work for older versions. All upgrades to this plugin will be completely backwards-compatible with previous versions, of course.</li>
    <li style="margin-top:0.5rem;">As mentioned above, if you want to use this plugin to encrypt/decrypt an image, or any other type of file, then you'd first convert the image into a <a href="https://developer.mozilla.org/en-US/docs/Web/URI/Schemes/data" target="_blank">data URL</a> (i.e. a text representation of the image/file).</li>
    <li style="margin-top:0.5rem;">If you want to decrypt something, but you've lost your private key, I'm afraid it's gone forever. A trillion dollars and a personal army of PhDs from the NSA's math department would probably not be enough to get your data back.</li>
    <li style="margin-top:0.5rem;">You might have noticed that if you encrypt the same text multiple times, you get a different result each time. This is important because if this were <u>not</u> the case, then if someone sent you an encrypted message on a public chat and the original message was e.g. "i love you", then if someone <i>else</i> also sent "i love you" to you, then the second person might notice that the encrypted text of their message is exactly the same as the encrypted message of the pervious person, so they'd know what the previous person said! And they're not meant to know that - it's meant to be completely private. So what this plugin does to prevent that, is to basically add some random data to the text that you encrypt, and then removes it during decryption. So you never need to worry about that random data, but it's there behind the scenes ensuring that even if you encrypt the same text multiple times, the encrypted text will come out different, so it's not possible to work out what someone else's message says via a method like in the above "i love you" example.</li>
    <li style="margin-top:0.5rem;">Using the <a href="https://perchance.org/send-me-a-secret-message" target="_blank">send-me-a-secret-message</a> example, you could actually create a "private group chat" between you and your friends by sharing a private key with one another. I.e. each of you creates your own "send-me-a-secret-message" page, like the one linked above. Then one of you ("Alice", say) creates a <b>new</b> public+private pair, and shares the public+private key with all the others via their "send-me-a-secret-message" page <i style="opacity:0.6;">(Wait, what?! Shares the public AND private key?! ... ← This is a good intuitive reaction. You're right to be cautious! You should never share your own <u>personal</u> private key. In this case Alice is creating a public+private key pair for the <u>whole group</u>. And she's sharing it with them via a <u>secure</u> method, so it's safe. But be careful when doing tricky/unusual stuff like this!)</i>. Then Alice also creates a new "send-me-a-secret-message" page that will act as the "private group chat" and embeds the group's <u>public</u> key in that page. Now everyone has the group's private key, so they can paste it in the private key box, and everyone starts chatting. Everyone's comments will be encrypted with the embedded public key (since that's what the code on the "send-me-a-secret-message" page does when you submit a comment), and the group members can see what each other are saying because they all have private key that's needed to use the "decrypt comments" button.</li>
    <li style="margin-top:0.5rem;">Check out more plugins at <a href="/plugins">perchance.org/plugins</a></li>
  </ul>
