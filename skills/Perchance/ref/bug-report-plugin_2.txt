<h1>Bug Report Plugin</h1>
<div style="margin:0 auto; width:600px; max-width:100%; text-align:left;">
  <p>This plugin is not well-documented, so it's mainly for use by more experienced Perchance devs. It helps you collect browser debug info for bug report / feedback submission, since most users won't know how to e.g. open the console and look for errors, or know their exact browser version, or the amount of localStorage they have available, and so on.</p>
  <p>A key feature of this plugin is that it encrypts the debug data and sets an expiry on it, so it can be safely submitted to a public <a href="/upload-plugin" target="_blank">comments-plugin</a> channel, for example.
  <p>It uses the <a href="/secret-plugin" target="_blank">secret-plugin</a> and the <a href="/upload-plugin" target="_blank">upload-plugin</a>, but it <a href="/dynamic-import-plugin" target="_blank">dynamically imports</a> them since most users of your generator won't be submitting bug reports or feedback, so we don't want to slow down initial page load by importing things that most users won't need. We'd rather take a bit longer during bug-report submission instead (due to having to dynamically load the plugins during the submission process).</p>
  <p>To use this plugin, you need to generate a public/private key pair using this tool: <a href="https://perchance.org/public-key-encryption-tool" target="_blank">perchance.org/public-key-encryption-tool</a>, as explained on the <a href="/secret-plugin" target="_blank">secret-plugin</a> page.</p>
  <p>All changes made to this plugin will be backwards-compatible, so you're free to use it in your generators.</p>

  Example 1: <a href="https://perchance.org/bug-report-plugin-encryption-example#edit" target="_blank">perchance.org/bug-report-plugin-encryption-example</a>
  <br><br>
  Example 2: <a href="https://perchance.org/bug-report-plugin-encryption-simpler-example#edit" target="_blank">perchance.org/bug-report-plugin-encryption-simpler-example</a>
  
  <p style="opacity:0.6; font-style:italic;">(Fun fact: The original reason I created the <a href="/secret-plugin" target="_blank">secret-plugin</a> was so that I could add the encryption feature to this plugin, since I wasn't happy with the fact that browser versions were being publicly exposed in the bug report data.)</p>
