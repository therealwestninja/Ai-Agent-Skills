Image Layer Combiner Plugin</h1>

<div style="margin:0 auto;width:100%;max-width:800px;background:white;border-radius:2px;padding:1em; box-sizing:border-box;">
  
  <p style="background:#ffd597; padding:0.5rem; border-radius:3px; margin-top:0;"><b>NOTE</b>: I previously recommended uploading your images to Imgur, but <a href="https://www.reddit.com/r/TheoryOfReddit/comments/12tinfp/imgur_has_announced_that_they_will_be_removing/" target="_blank" style="color:red;">they've started deleting uploaded images</a>. Luckily we now have <a href="https://perchance.org/upload" target="_blank" style="font-weight:bold;">perchance.org/upload</a>, so use that instead, or all your hard work may be destroyed by Imgur at some point. Also, I <u>highly</u> recommend saving images in <b>webp</b>✅ format instead of <b>png</b>❌, since webp file sizes can be as much as 10x smaller for the same quality, and they support transparency and animations.</p>
  
	<p style="text-align:justify; margin-top:0;">Here's how this plugin works: You give it a set of images for each "layer" (e.g. head, body, etc. in the example below), and it'll select one random image from each layer and overlay them to produce a single final image. To use this plugin, you'll first need to import the it by putting this code somewhere in your Perchance code panel:</p>
	 
<pre class="custom" style="margin:0; height:100%; box-sizing:border-box;">
imageLayerCombiner = \{import:image-layer-combiner-plugin\}
</pre>
	
  <p>Then you need to specify the images that are in each layer, and optionally add <code>height</code> or <code>width</code> settings like so:</p>
	
<pre class="custom" style="margin:0; height:100%; box-sizing:border-box;">
data
	settings
		height = 200px
  layers
		hat
			https://i.imgur.com/1spyUp1.png
			https://i.imgur.com/snpdUvc.png
	  eyes
			https://i.imgur.com/PShX0T8.png
			https://i.imgur.com/j6WWWPO.png
		head
			https://i.imgur.com/ur3TABt.png
			https://i.imgur.com/ERbhMDS.png
		...
</pre>

	<p>and then in the HTML panel (bottom-right panel) you write this:</p>
	
<pre class="custom" style="margin:0; height:100%; box-sizing:border-box;">
\[imageLayerCombiner(data)\]
</pre>
	
	<p>And that would produce something like this (please forgive my drawings...):</p>
	
	<p id="example1" style="padding:1rem; background:#efefef; text-align:center;">[$output(exampleData)]</p>
	<button onclick="update(example1)">randomize</button>
	
	<p>You can <b>double-tap or right-click on the image</b> to open a full-resolution version in a new browser tab. Then just right-click and click "Save image..." to <b>download the generated image</b>. <a href="https://perchance.org/image-layer-combiner-example#edit" target="_blank">Here's an example generator</a> with the full code of the above example.</p>
	
	<p><b>Note</b>: Each image should have exactly the same dimensions as one another because they will be overlayed on top of one another without any special positioning. For example, here are all the images used in the above example:</p>
	
	<div>[exampleImagesHtml]</div>
	
	<p>Notice that each image takes up the full "canvas", even though there's lots of blank space. You should make your images just like that.</p>
  
  <div style="background:#ddd; padding: 0.9rem 1rem;">
    <p style="margin-top:0;"><b>Side note</b>: I've noticed some people using this plugin are creating many versions of the same image - one for each color. Note that you don't need to do that - you can create one version (e.g. a red one) and then add the special <code>filter</code> property to the layer to randomly change the hue:</p>
  <pre class="custom" style="margin:0; height:100%; box-sizing:border-box;">
  layers
    hat
      filter = hue-rotate(\{0-360\}deg) <span style="opacity:0.5;">// &lt;-- this randomizes colors of this layer</span>
      https://i.imgur.com/1spyUp1.png
      https://i.imgur.com/snpdUvc.png
    ...
  </pre>
    <p style="margin-bottom:0">There are lots of other filter types (e.g. blur, saturation, brightness), and you can combine them together. Play with the filters <a href="https://perchance.org/css-filter-effects" target="_blank">here</a>, and see an example of this plugin with filters <a href="https://perchance.org/filter-image-combiner-example#edit" target="_blank">here</a>.</p>
	</div>
  
	<p>The easiest way to create a set of images like the one in the above example is to first draw a "base" image which acts as your template for all the different parts of the image. Make this background layer somewhat transparent so it doesn't get in the way of your drawings. Then add a new layer in your image-editing app and draw one of the pieces (e.g. a head), using the background template as a guide. Once you've finished drawing that part, hide the background layer and export/save the image as a PNG and upload it somewhere (e.g. <a href="https://catbox.moe">catbox.moe</a>, or <a href="https://perchance.org/upload">perchance.org/upload</a>), and then un-hide the background repeat that process for each of the parts of each of your layers.</p>
	
	<p>If you're not sure what image editing app to use, I recommend <a href="https://photopea.com" target="_blank" rel="noopener noreferrer">Photopea</a>. It's an excellent browser-based editor that's similar to Photoshop.</p>
	
	<p>Note that you can select a bunch of images and once and drag-and-drop them all to <a href="https://catbox.moe">catbox.moe</a>, or <a href="https://perchance.org/upload">perchance.org/upload</a> to upload them all at once.</p>
<!--   <p>Note: Make sure you're using the *direct* link to the image by right-clicking the image and then selecting "Copy image address" or "Copy image location" or something like that (depending on which browser you're using). If the copied link starts with "blob:" that just means you need to wait a little bit longer for it to properly upload before right-clicking.</p> -->
	
	<p>Note that you can call the layers whatever you want. Their names are irrelevant - they're just labels to help you remember which sets of URLs refer to which parts of you image. The first layer in the list will be displayed on top of the next one in the list, and so on. The last layer will be displayed at the bottom.</p>
	
	<p>Also: You should <b>make sure to save your images as PNGs</b> or another type that allows transparency. If you use JPEG images, then all you'll see is the top layer because it will cover up the ones below - the JPEG type doesn't support transparency.</p>
	
	<p>I recommend labelling your image links with comments like in the following example. Comments (text preceded by two forward-slashes) are ignored by Perchance, so they're purely to help you annotate your Perchance code with helpful notes.</p>
	
<pre class="custom" style="margin:0; height:100%; box-sizing:border-box;">
layers
  hat
	  https://i.imgur.com/1spyUp1.png  <span style="opacity:0.4;">// backwards cap</span>
		https://i.imgur.com/snpdUvc.png  <span style="opacity:0.4;">// top hat</span>
	eyes
		https://i.imgur.com/PShX0T8.png  <span style="opacity:0.4;">// big</span>
		https://i.imgur.com/j6WWWPO.png  <span style="opacity:0.4;">// small</span>
	...
</pre>
	
	<p>The reason labelling is helpful is because you'll often need to redraw certain parts every now and then, and it's annoying to try to find which url corresponds to which drawing. Without labels you'd have to open each URL one-by-one until you find the drawing that you wanted to replace.</p>
	
  <p>By default a random item is selected from each layer with <code>selectOne</code>, but if you wanted each layer to be a <a href="https://perchance.org/consumable-list-loop-plugin" target="_blank">"looping" consumable list</a> (so it's "re-shuffled" when it runs out), then you could do that like this:</p>
<pre class="custom" style="margin:0; height:100%; box-sizing:border-box;">
consumableListLoop = \{import:consumable-list-loop-plugin\}

data
  layers
    hat
      $output = \[this.cLoop = this.cLoop || consumableListLoop(this)\]
      https://i.imgur.com/1spyUp1.png
      https://i.imgur.com/snpdUvc.png
      ...
    ...
</pre>
  <p>So as you can see you just need to add that <code>$output</code> line to the top of each of your layer lists and there'll be a much smaller chance of the same item being selected twice in a row (note that it is still possible though - for the same reason that it's possible that the last card you draw from a deck is the same as the first card you pick after reshuffling it).</p>
  
	<p><b>Extra notes:</b></p>
	<ul class="dot-list">
    <li>You can call the layers whatever you want - they don't need to be called "hat", "eyes", etc.</li>
		<li>Use <a href="https://perchance.org/image-layer-combiner-example#edit">this example</a> as a guide to get started.</li>
		<li>If you want to allow the users of your generator to <b>"lock" certain parts of the image</b> and only randomize the others, you can do that by using the <a href="/locker-plugin" target="_blank">locker-plugin</a> as shown in <a href="https://perchance.org/lockable-image-combiner-example#edit" target="_blank">this example generator</a>.</li>
		<li><a href="https://perchance.org/image-layer-combiner-thumbnail-generator-example#edit" target="_blank">Here's an example generator</a> that generates thumbnails <b>based on</b> previously selected list items / variables.</li>
    <li><a href="https://perchance.org/image-layer-combiner-with-variable-conditions#edit" target="_blank">Here's another example</a> that uses simple variable conditions.</li>
    <li><a href="https://perchance.org/image-layer-combiner-checkbox-example#edit" target="_blank">Here's another example</a> that has user-inputs (checkboxes) to disable/enable certain categories of images in certain layers.</li>
		<li>Want to <b>randomize the colors in your images</b>? Or randomly add other effects like brightness, contrast, etc.? You can use a the special <code>filter</code> sub-property on layers and URLs as shown/explained in <a href="https://perchance.org/filter-image-combiner-example#edit">this example generator</a>.</li>
		<li>You can of course use GIF images if you want your layers to be animated (but you'll need to make sure your GIFs have a transparent background).</li>
		<li>If you want to add custom CSS to the container of the images, you can use the <code>containerCss</code> property in the <code>settings</code> sub-list.</li>
		<li>If you add only the <code>height</code> property in the <code>settings</code>, then the width of the image will automatically change to keep the correct proportions. And the same will happen if you only set the width (height will adjust automatically). If you set both the width and the height, then your image may become distorted unless you set the correct values to maintain the original aspect ratio of the images.</li>
		<li>In the above example's I've called the list that holds the settings and layers "<code>data</code>", but you can call it whatever you want. For example, if you called it <code>imageData</code>, then you'd write <code>\[imageLayerCombiner(imageData)\]</code> in your HTML panel.</li>
		<li>Check out more plugins at <a href="/plugins">perchance.org/plugins</a></li>
	</ul>
</div>
